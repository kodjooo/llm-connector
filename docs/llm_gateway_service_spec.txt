ТЗ: thin relay сервис для OpenAI

1. Назначение
Сервис размещается на втором сервере в разрешенной юрисдикции и нужен только для сетевого доступа к OpenAI API. Он не хранит бизнес-правила классификации, не формирует prompt и не интерпретирует ответ модели.

2. Endpoint
POST /v1/openai/chat-completions

3. Авторизация
- Заголовок: Authorization: Bearer <token>
- Токен хранится в relay как `GATEWAY_AUTH_TOKEN`
- Без корректного токена сервис возвращает 401

4. Входной payload
Relay принимает OpenAI-совместимый JSON для `chat/completions`.
Обязательные поля:
- `messages`
Опциональные поля:
- `model`
- `response_format`
- и другие допустимые поля OpenAI

Пример:
{
  "model": "gpt-5-mini",
  "response_format": {
    "type": "json_schema",
    "json_schema": {
      "name": "SiteClassification",
      "schema": {
        "type": "object",
        "properties": {
          "site_verdict": {"type": ["string", "null"]},
          "detected_city": {"type": ["string", "null"]},
          "confidence": {"type": "number"},
          "reason": {"type": ["string", "null"]}
        },
        "required": ["site_verdict", "detected_city", "confidence", "reason"]
      }
    }
  },
  "messages": [
    {"role": "system", "content": "Определи тип сайта по контексту и верни только JSON."},
    {"role": "user", "content": "{\"domain\":\"verno.pro\",\"expected_city\":\"Краснодар\"}"}
  ]
}

5. Выходной payload
Relay возвращает raw JSON от OpenAI `chat.completion` без преобразования в доменные поля.

6. Ошибки
- 400: невалидный payload
- 401: неверный токен
- 429: локальный rate limit
- 502: upstream error / invalid upstream response
- 504: upstream timeout

7. Ограничения и защита
- Не логировать Bearer token
- Не логировать полный prompt
- Ограничивать размер payload
- Поддерживать allowlist IP

8. Эксплуатация
- Healthcheck: GET /health
- Readiness: GET /ready
- Метрики: GET /metrics
- Retry/backoff к OpenAI через env-конфиг

9. Критерий приемки
- Основной pipeline может отправить в relay тот же payload, который раньше отправлял прямо в OpenAI
- Relay успешно проксирует запрос и возвращает OpenAI-совместимый ответ
- Бизнес-проект сам парсит этот ответ без дублирования prompt-логики на втором сервере
