LLM Gateway Service Spec

Purpose:
- Accept a ready-to-send OpenAI Responses API payload from the main pipeline.
- Forward it to OpenAI from an allowed jurisdiction.
- Return the raw OpenAI JSON response unchanged.

Endpoint:
- `POST /v1/openai/responses`

Authorization:
- Require `Authorization: Bearer <GATEWAY_AUTH_TOKEN>`.

Request contract:
- Accept a nearly transparent OpenAI Responses API payload.
- Require at least `input`.
- Allow optional `model`; if omitted, use `DEFAULT_OPENAI_MODEL`.
- Pass through extra OpenAI fields as-is.

Expected payload example:
```json
{
  "model": "gpt-5-mini",
  "text": {
    "format": {
      "type": "json_schema",
      "name": "SiteClassification",
      "schema": {
        "type": "object",
        "properties": {
          "site_verdict": {"type": ["string", "null"]},
          "detected_city": {"type": ["string", "null"]},
          "confidence": {"type": "number"},
          "reason": {"type": ["string", "null"]}
        },
        "required": ["site_verdict", "detected_city", "confidence", "reason"]
      }
    }
  },
  "input": [
    {
      "role": "system",
      "content": [
        {"type": "input_text", "text": "Return only JSON."}
      ]
    },
    {
      "role": "user",
      "content": [
        {"type": "input_text", "text": "{\"domain\":\"verno.pro\"}"}
      ]
    }
  ]
}
```

Response contract:
- Return raw OpenAI `response` JSON.
- Do not add business-level normalization.

Error codes:
- `400` invalid_request
- `401` unauthorized
- `429` rate_limit_exceeded
- `502` upstream_invalid_response
- `504` upstream_timeout
